## What XCore Plugin provides

<CardGroup cols={2}>
  <Card title="Player management" icon="user">
    Persistent player profiles with PvP ratings, Hexed progression, playtime tracking, and a rich in-game stats menu.
  </Card>
  <Card title="Moderation" icon="shield">
    Ban, mute, votekick, and temporary-ban tools with cross-server broadcast over Redis streams.
  </Card>
  <Card title="Game modes" icon="gamepad">
    Built-in support for Hexed, PvP, Last Standing, and Event game modes with per-mode statistics.
  </Card>
  <Card title="Badges" icon="award">
    Unlockable player badges (Developer, Translator, Veteran, and more) displayed in chat and player profiles.
  </Card>
  <Card title="Private messages" icon="envelope">
    Persistent cross-server private messaging with inbox, reply, and player-block flows.
  </Card>
  <Card title="Chat translation" icon="language">
    Configurable translation pipeline (Google Translate + OpenAI-compatible providers) for multilingual servers.
  </Card>
  <Card title="Discord integration" icon="discord" href="/features/discord-integration">
    Link player accounts to Discord, receive moderation logs, and manage admin access from Discord.
  </Card>
  <Card title="Cross-server networking" icon="network-wired" href="/architecture/networking">
    Redis-backed pub/sub and RPC layer for global chat, moderation events, and server discovery.
  </Card>
</CardGroup>

## Quick start

<Steps>
  <Step title="Install dependencies">
    Ensure you have Java 25+, MongoDB 6.0+, Redis, and Mindustry v155.4+ installed on your server.
  </Step>
  <Step title="Build the plugin">
    Clone the repository and run `./gradlew shadowJar` to produce the plugin JAR.
  </Step>
  <Step title="Deploy to Mindustry">
    Copy the JAR from `build/libs/` to your server's `config/mods/` directory.
  </Step>
  <Step title="Configure connections">
    Edit `config/mods/XCore/servers.json` with your MongoDB connection string and Redis URL, then restart the server.
  </Step>
</Steps>

<Note>
  See the [Installation](/installation) and [Configuration](/configuration) pages for complete setup instructions.
</Note>
